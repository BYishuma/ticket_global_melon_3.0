# Melon Ticket 自动抢票插件 

一个用于 Melon Global Ticket 网站的自动抢票浏览器扩展插件，支持自动选座、邮件提醒和弹窗自动处理功能。

## 功能特性

- ✅ **自动选座**：智能选择最佳座位（连座优先、前排优先）
- ✅ **邮件提醒**：找到票时自动发送邮件通知
- ✅ **弹窗自动处理**：自动处理 "Please try again." 错误弹窗
- ✅ **人性化操作**：模拟真实用户行为，避免被检测
- ✅ **多区域支持**：支持多个区域自动搜索

## 安装步骤

### 1. 下载代码

```bash
git clone https://github.com/your-username/ticket_global_melon_3.0_ManifestV3.git
cd ticket_global_melon_3.0_ManifestV3
```

### 2. 加载到 Chrome 浏览器

1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 开启右上角的"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择项目文件夹

### 3. 配置邮件提醒（可选）

#### 步骤 1：部署 Google Apps Script

1. 访问 [Google Apps Script](https://script.google.com)
2. 点击"新建项目"
3. 复制项目中的 `google-apps-script-code.gs` 文件内容到代码编辑器
4. 点击"部署" → "新建部署"
5. 选择类型为"Web 应用"
6. 配置如下：
   - **执行身份**：选择"我"
   - **具有访问权限的用户**：选择"所有人"
7. 点击"部署"并完成授权
8. 复制生成的 **Web 应用 URL**

#### 步骤 2：在代码中配置邮箱

打开 `content/email.js` 文件，修改以下配置：

```javascript
const DEFAULT_EMAIL_CONFIG = {
    scriptUrl: 'YOUR_GOOGLE_APPS_SCRIPT_URL_HERE',  // 替换为您的 Web 应用 URL
    recipientEmail: 'your-email@gmail.com'  // 替换为您的邮箱地址
};
```

保存文件后，重新加载插件即可生效。

## 使用方法

### 1. 添加抢票任务

1. 点击浏览器工具栏中的插件图标
2. 点击"添加新任务"按钮
3. 填写以下信息：
   - **演唱会名称**：输入演唱会名称（可选）
   - **演唱会ID**：输入6位数字演唱会ID
   - **购票数量**：选择需要购买的票数（1-4张）
   - **日期**：选择演出日期
   - **开演时间（KST）**：选择开演时间
   - **拟选区域**：输入区域名称，多个区域用逗号分隔（如：A,B,C）
   - **新增必要延迟(ms)**：设置延迟时间（可选，默认0）

4. 点击"保存"

### 2. 开始抢票

1. 访问 Melon Global Ticket 网站
2. 进入目标演唱会的购票页面
3. 插件会自动开始搜索和选座

### 3. 监控抢票状态

- 打开浏览器控制台（F12）查看详细日志
- 找到票时会收到邮件通知（如果已配置）
- 浏览器会显示桌面通知

## 功能说明

### 自动选座策略

插件采用智能选座算法：

1. **连座优先**：优先选择连续的座位
2. **前排优先**：在连座的基础上，优先选择前排座位
3. **同排选座**：如果找不到连座，选择同排的座位
4. **跨排选座**：最后选择最近的可用座位

### 弹窗自动处理

插件会自动检测并处理以下弹窗：

- "Please try again." 错误弹窗
- 自动在弹窗显示前拦截，无需手动操作

### 邮件提醒

当插件检测到有票可用时，会自动发送邮件到您配置的邮箱地址。邮件包含：

- 演唱会信息
- 日期和时间
- 选择的区域
- 检测时间

## 故障排除

### 插件无法加载

- 检查是否开启了"开发者模式"
- 确认选择了正确的项目文件夹
- 查看 `chrome://extensions/` 页面是否有错误提示

### 无法自动选座

- 检查控制台是否有错误信息
- 确认演唱会ID和区域名称是否正确
- 检查网络连接是否正常

### 邮件未收到

- 检查 Google Apps Script 是否已正确授权
- 确认 Web 应用 URL 配置正确
- 检查垃圾邮件文件夹
- 查看控制台是否有错误信息

### 弹窗无法自动关闭

- 检查控制台是否有 "✅ 已拦截 iframe alert" 日志
- 确认弹窗类型是否为浏览器原生弹窗
- 查看是否有其他错误信息
